# 实践小项目 - P87-P91 武器奖励与可击毁箱子

## 教程/小节名称

唐老狮 Unity 教程 - 实践小项目 P87 到 P91

## 本节目标

继续推进 GameScene 核心玩法，完成玩家小地图、武器和子弹、奖励拾取、属性变化以及可击毁箱子/墙体。

## 已完成内容

- P87：完成玩家小地图。
  - 没有照搬教程 GUI 写法。
  - 使用 `RenderTexture`、小地图摄像机和 UGUI `RawImage` 实现。
  - 小地图摄像机改为正交相机，并放在玩家正上方，让玩家稳定处于小地图中心。
- P88：完成武器和子弹对象。
  - 新增 `WeaponObj`，负责从发射点生成子弹。
  - 新增 `BulletObj`，负责子弹移动、命中墙体后的销毁和命中特效。
  - 子弹音效继续读取设置中的音效开关和音量。
- P89：完成武器奖励对象和获取特效。
  - 新增 `WeaponReward`，玩家拾取后随机切换武器。
  - 拾取后生成特效并播放音效。
- P90：完成属性奖励对象。
  - 新增 `PropReward`，支持攻击、防御、生命值、最大生命值等奖励类型。
  - 生命值和最大生命值变化时同步到 `GamePanel` 血条。
- P91：完成可击毁箱子/墙体。
  - 新增 `BreakableWallObj`，被触发后播放销毁特效并销毁自身。
  - 按概率从奖励数组中生成奖励。
  - 在 `TagManager` 中补充 `UnbreakableWall` 和 `BreakableWall`。
- 新增 `AutoDestroy`，用于让特效对象在一定时间后自动销毁。

## 做得好的地方

- 能根据当前项目技术路线选择更合适的小地图方案，没有被教程旧式 GUI 写法限制住。
- 武器、子弹、奖励和可击毁箱子逐渐拆成独立对象和脚本，职责清晰度比集中写在玩家脚本里更好。
- 遇到奖励触发器被非玩家对象触发导致空引用后，能用 `CompareTag("Player")` 先过滤触发对象。
- 继续复用前面做好的音乐/音效设置数据，让子弹、奖励和特效音效受全局设置控制。

## 当前问题

- `BreakableWallObj` 当前没有显式判断触发者是不是子弹；如果后续出现非子弹对象误触发，需要补 `Bullet` Tag 或 `BulletObj` 组件判断。
- `WeaponObj.Fire()` 依赖多个 Inspector 引用：`bulletPrefab`、`bulletSpawn`、子弹上的 `BulletObj` 和 `AudioSource` 都要正确配置。
- `WeaponReward` 和 `PropReward` 已过滤 `Player` Tag，后续要确认玩家对象始终带 `Player` Tag。
- `PlayerObj.OnDisable()` 中 `playerAction` 的空判断问题仍可以在后续收尾时修正。

## 推荐改进方向

- P92 开始接入敌人后，子弹命中逻辑会从“只处理墙体”扩展到“处理敌人/箱子/墙体”，可以考虑把伤害处理拆得更清楚。
- 如果后续奖励种类继续增加，可以把奖励基类或接口抽出来，减少 `PropReward` / `WeaponReward` 重复的特效和音效代码。
- 当前小项目阶段继续保持能跑优先；收尾时再统一补空判断、Tag 判断和 Inspector 配置检查。
