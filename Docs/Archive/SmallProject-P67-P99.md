## P77 音效数据逻辑记录

### 当前实现

- 使用 UGUI 制作设置窗口。
- 设置窗口关闭按钮触发保存逻辑。
- 数据通过 `GameDataMgr` 管理，再交给 `PlayerPrefsDataMgr` 保存到 PlayerPrefs。
- 音乐开关、音效开关、音乐音量、音效音量等数据放在 `MusicData` 中。
- P77 已完成，设置数据保存链路已经跑通。

### 本次理解点

`MusicData` 里写字段默认值，例如：

```csharp
public bool isFirst = true;
```

创建对象时确实会先执行，`isFirst` 在那一瞬间是 `true`。

但当前项目里 `musicData` 不是直接使用 `new MusicData()` 的结果，而是使用：

```csharp
musicData = PlayerPrefsDataMgr.Instance.LoadData(typeof(MusicData), "Music") as MusicData;
```

`LoadData()` 内部会先创建对象，再遍历所有字段，用 PlayerPrefs 读取结果重新赋值：

```csharp
fieldInfo.SetValue(data, LoadValue(fieldInfo.FieldType, loadKeyName));
```

所以字段默认值会被 `PlayerPrefsDataMgr` 的加载结果覆盖。

### 问题原因

当 PlayerPrefs 中还没有某个 bool key 时：

```csharp
PlayerPrefs.GetInt(keyName)
```

默认返回 `0`。如果加载 bool 时用 `GetInt(keyName) == 1` 判断，那么不存在的 key 会被当作 `false`。

因此，即使 `MusicData` 中写了 `isFirst = true`，也会在 `LoadData()` 中被覆盖成 `false`。

### 经验总结

- C# 字段默认值会先执行，但最终 `musicData` 的内容取决于 `PlayerPrefsDataMgr.LoadData()` 的读取规则。
- 当前工具类分不清“PlayerPrefs 中没有这个 key”和“这个 key 存在但值是 false / 0”。
- 如果使用 `notFirst` 标记，第一次初始化默认数据后要把 `notFirst` 设置为 `true` 并保存。
- 更稳的写法是结合 `PlayerPrefs.HasKey()` 判断是否已有存档，或在 key 不存在时保留对象字段默认值。
