# 实践小项目 P78-P79：排行榜界面与数据逻辑

## 教程/小节名称

唐老狮 Unity 教程 - 实践小项目 P78「排行榜界面」与 P79「排行榜数据逻辑」

## 本节目标

完成开始场景中的排行榜窗口和排行榜数据逻辑，让玩家成绩可以被添加、排序、截断、显示并保存到本地。

## 已完成内容

- 使用 UGUI 制作排行榜窗口。
- 通过 `RankPanel` 获取排名、姓名、分数和时间文本，并在显示面板时刷新排行榜内容。
- 新增 `RankInfo` 保存单条成绩：玩家名、用时、分数。
- 新增 `RankInfoList` 保存排行榜列表，配合自定义 `PlayerPrefsDataMgr` 做本地持久化。
- 在 `GameDataMgr.AddRankInfo()` 中实现添加成绩、判断同名、排序、保留前 9 条和保存。
- 将教程中只按时间排序的写法调整为当前项目更合理的业务规则：分数高的排前面，分数相同才比较时间，用时短的排前面。
- 对同一个 Name 重复提交成绩的情况加入覆盖逻辑：只保留该 Name 的最好成绩。

## 做得好的地方

- 没有机械照搬教程排序写法，而是先思考排行榜应该表达什么规则。
- 能识别 `Sort` 比较器返回值的含义：负数表示第一个对象排前，正数表示第二个对象排前，`0` 表示排序上相等。
- 能主动补上教程没有处理的同名成绩覆盖逻辑，避免同一个玩家在排行榜中出现多条记录。
- 能发现并修正“新高分只覆盖分数、不覆盖时间”带来的数据拼接问题。

## 当前问题

- `RankPanel.Start()` 中仍有用于验证排行榜的测试数据添加逻辑，后续接入正式结算流程时需要删除或迁移到专门的调试入口。
- 当前排行榜 UI 和数据截断都按 9 条处理；如果后续界面改成 10 条，需要同步修改循环范围和截断条件。
- 时间显示目前是简单的 `小时:分钟:秒` 字符串拼接，暂未做补零格式化。

## 推荐改进方向

- 后续在通关或失败结算界面中调用 `AddRankInfo()`，用真实玩家名、分数和通关时间写入排行榜。
- 可以把时间格式改成 `00:01:05` 这种固定宽度格式，让排行榜更整齐。
- 如果后续允许重名玩家同时存在，可以把去重依据从 `name` 升级为玩家 ID；当前小项目阶段用 Name 去重已经足够。
